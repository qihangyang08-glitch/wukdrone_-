

### **工作日志：WuK无人机电控系统开发 - 第一阶段**

**日期：** 2025年11月26日
**负责：** 电控组
**事项：** 架构规划与仿真环境搭建

#### **一、 系统架构重构 (System Architecture Design)**
1.  **控制策略升级：**
    *   确立了 **WuK V2 新版控制方案**：摒弃原有的“Arduino 外挂逻辑”，转向 **ArduPilot 源码级深度开发**。
    *   目标：实现 **Mid-air Morphing（空中变构）** 与 **一键自动着陆** 功能。
2.  **变构逻辑定义：**
    *   明确了 **“变几何下蹲 (Variable Geometry Squat)”** 的机械-控制耦合逻辑。
    *   制定了 **“空中变构 + 动力切断 + 惯性着陆”** 的安全控制策略，规避MPC算法算力不足的风险，采用 **动态混控 (Dynamic Mixing)** 方案。

#### **二、 开发环境部署 (Development Environment Setup)**
1.  **Linux 子系统搭建：**
    *   基于 Windows 11 部署 **WSL2 (Ubuntu 20.04/22.04)**，构建了原生 Linux 编译环境。
    *   完成 **系统盘迁移**：将 WSL 文件系统（ext4.vhdx）从 C 盘迁移至 D 盘，解决了源码编译带来的存储空间瓶颈。
    *   配置 **默认用户权限**：修复了系统迁移导致的用户丢失问题，恢复 `qwssx2` 为默认登录用户。
2.  **源码版本控制初始化：**
    *   **工具链部署：** 在 Linux 环境下安装独立 Git 工具，避免了 Windows/Linux 换行符（CRLF/LF）冲突。
    *   **网络环境配置：** 解决了 WSL2 与 Windows 宿主机之间的网络隔离问题，成功配置 **Git 代理/镜像通道**，打通 GitHub 访问权限。
    *   **源码获取：** 成功克隆 **ArduPilot Master 分支** 源码，为后续开发最新 H743 驱动和自定义物理模型奠定基础。
